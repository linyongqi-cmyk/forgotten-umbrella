## 一、先记住三个词（最重要）

把项目想象成一份**正在写的作业**：

| 词 | 大白话 | 谁能看到 | 风险 |
|----|--------|----------|------|
| **本地修改** | 你在电脑上改文件 | 只有你 | 无 |
| **存档 (commit)** | 给当前状态拍张快照，能随时退回 | 只有你（仍在电脑里） | 无 |
| **上线 (push + 发布)** | 把项目传到网上，做成网页 | 全世界 | 半成品会被看到 |

**关键：电脑里的修改不会自己跑到网上。必须你主动说“上线”，Claude 帮你 push，才会发布。**

## 三、日常开发的循环（每次想改东西时）

1. **告诉 Claude 你想改什么**（比如“把欢迎页的标题改成 XXX”）。
2. Claude 改完后，**打开预览看效果**：
   - 地址：`http://127.0.0.1:4173/`
   - 如果打不开，就跟 Claude 说“帮我启动预览服务器”。
3. 满意 → 让 Claude **“做一次存档”**。
4. 不满意 → 继续说怎么改，或让 Claude **“退回上一次存档”**。

---

## 四、什么时候该「存档 (commit)」？

**原则：完成一个小目标就存一次。** 存档免费、零风险、越勤越安全。

该存档的时机举例：
- 改完了某个页面的一块内容（比如关于页的文案）。
- 调好了某个交互/动画。
- 加了几条新的伞记录。
- **每次结束今天的工作前**，一定存一次。

怎么做：直接说 **“帮我存档，备注是‘XXX’”**（备注就是一句话说明这次改了啥，方便以后查）。

---

## 五、什么时候该「上线 (push + 发布)」？

**等以下都满足了，再考虑第一次上线：**
- [ ] 主要页面（地图 / 档案 / 关于）的设计和文案基本定稿。
- [ ] 主要交互都正常，没有明显 bug。
- [ ] 你愿意让别人看到这个版本。

到那时，跟 Claude 说 **“我准备上线了”**，它会带你走这几步：
1. 在 GitHub 创建仓库（你项目的网上文件夹）。
2. 把项目 push 上去。
3. 开启 GitHub Pages，拿到一个公开网址。

之后再改动，只要说 **“同步到线上”**，最新版就会自动更新到网站（项目里已经配好自动部署）。

## 六、编辑「伞记录」数据的专门流程

伞的数据**真正的源头**在 `filebox/records/` 文件夹里，每把伞一个小文件夹，里面有：
- 一张图片
- 一个 `record.json`（记这把伞的地点、时间、类型、故事等）

⚠️ **改完 record.json 后，必须重新“生成数据”**，网页才会更新。
- 跟 Claude 说 **“重新生成伞的数据”**（它会运行 `npm run records:build`）。
- 网页真正读取的是 `data/umbrellas.json`，这个文件是**自动生成的，不要手改**。

## 八、几条安全须知

- **图片别乱删**：`filebox/records/` 里的图是网站要用的，删了网页会缺图。
- **`filebox/choice/` 是旧备份**，已设置成不上传，可以不管它。
- **API Key 已配置好**，地图能正常显示，不用动。
- 拿不准的操作，**先问 Claude “这样做有什么后果”**，再决定。
