## 历史待办（原文，已全部完成，留作参考）

### item 5 — 数量选 unknown 时的 object/state 规则
- **现象**：用户给某些图片（如 IMG_6859、IMG_6895）的"伞的数量"选了 `unknown`，导致编辑器规则不让填 object/state，详情页也不显示。
- **要做**：
  1. 数量=unknown 时，object 和 state 默认视为 `unknown` 并**在详情页显示**（INFORMATION 区 object/state 显示 unknown，而不是空白）。
  2. **统计交叉表计数规则**：数量=unknown 的记录，因为不知道有几把伞，**不计入交叉表数量**（buildStatsUnits 要跳过/排除它们）。
  3. **要区分**：有些图数量选了 `1` 但颜色/状态判断不了——这种 object、state 也是 unknown，**但要计入交叉表**（因为知道是 1 把）。
- **关键文件**：编辑器规则在 `app.js` 后半段（搜 `umbrellaCount`/`umbrellaUnits`/`renderEditor` 等）；详情页 INFORMATION 在 `renderFocusArticle`/`buildObjectGroups`/`statusLinesFromUnits`（`app.js`）；统计在 `buildStatsUnits`（`app.js` ~1533）。区分"数量unknown"vs"数量1但属性unknown"靠 `item.umbrellaCount === "unknown"`。
- 数据字段见 CLAUDE.md：`umbrellaCount` ∈ "1"~"5"/"unknown"/""，`umbrellaUnits[]` = `{color,colorDetail,kind,status[],statusOther}`。

### item 6 — 总览表 多伞分行 + 排序筛选
- **6a 多把伞分行**：一张图有多把伞时（如 IMG_8317、IMG_8056），总览每把伞各占一行，ID 写成 `IMG_8056(1)`、`IMG_8056(2)`…。按 `umbrellaUnits` 拆。注意和 item5 的计数规则配合（数量unknown的怎么显示一行）。
- **6b 排序/筛选**：给总览各列加筛选，参照アーカイブ现有按钮逻辑复用：
  - date：和アーカイブ「時間」按钮一样（升降序，`sortByTime`/`state.archiveOrder`）。
  - type、area：加类似筛选（参考 `renderArchiveSecondary`/`filterArchiveItems` 的子筛选 chip 思路）。
  - object、state：也加筛选。
- **关键文件**：`renderStatsOverview`（`app.js` ~1652）；可复用 `sortByTime`、`countByField`、アーカイブ的 secondary chip 模式。

### item 12 — 文字内容 JSON + 编辑模式（最大的一项）
- **目标**：把散在代码里的"文案"集中到一个 JSON 文件，做成**编辑模式**可改，**必须日英双语**。
  - 要纳入的文案：① 9 个 type 的说明文（现在是 `app.js` 里的 `TYPE_DESCRIPTIONS` 常量，~1411 行，`{type: {ja:[段落...], en:[段落...]}}`）；② 统计页说明文（现在是 `app.js` 的 `STATS_INTRO` 常量，**只有日语**，需补英文）。
  - 建议：新建 `data/texts.json`（或 `filebox/` 下），前端读取；本地编辑器加一个"文案编辑"面板，存回走 `scripts/editor-api.mjs` 的 `/api/*`（参考现有 save-record）。
  - **限制**（用户明确）：这个编辑模式**不要**像详情页那样的"加片段"功能（文字插哪儿不好简单操作），只做对现有文案的**修改**即可。
- **本地编辑器**只在 127.0.0.1 出现（`IS_LOCAL`），线上不渲染——文案编辑也照此。
- 改完别忘了：TYPE_DESCRIPTIONS / STATS_INTRO 改成从 JSON 读取后，把旧常量删掉。
